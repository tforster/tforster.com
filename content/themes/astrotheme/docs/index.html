Astro Theme

Astro is a content focused responsive theme for the Ghost blogging platform. Designed from the bottom up to be high performance, user friendly, and accessible on all platforms. Astro adapts to the users device, so it looks great on smartphones, tablets, TVs, and even the latest 4K monitors. Taking advantage of the ghost platform, Astro has two different post styles. The standard style is great for update posts and short articles, while featured posts have full page titles with image backdrops, and work great with large articles, tutorials and other documentation.

Installation

  • 1

    Locate the theme directory within the download files, it can be found at theme/Astrotheme.

  • 2

    Login to FTP/SFTP of your blog, or the file browser of your provided hosting service.

  • 3

    Navigate to the /content/theme directory within your ghost installation.

  • 4

    Upload the Astrotheme directory found in step one into the remote /content/theme directory.

  • 5

    Restart your Ghost service. The process for this will change dependent on your provider. If you are unaware how to restart the service please raise a support issue with your host.

  • 6

    Login to your Ghost admin panel. This can be found at yourdomain.com/ghost.

  • 7

    Navigate to the Settings > General tab.

  • 8

    In the Theme drop down, select the newly installed Astro Theme. Click Save to save the theme changes.

Customization

Blog Options

  • Featured Post Images
    Each featured post can have a feature image which is displayed within the header of the post page. More information on this feature and how to use it can be found below in the 'Features' section.
  • Posts Per Page
    For best theme visuals we recommend setting this to any odd number above 5.

Theme Options

These options can be found within the file /assets/js/global.min.js with included documentation. These are all located on the first line of the file. Replace the current values with your required settings.

  • Disqus Shortname
    Enter your Disqus account shortname here to enable Disqus comments for blog posts. Requires an account, which can be obtained for free at http://disqus.com. More information can be found below.
  • Google Analytics ID
    Enter your Google Analytics ID here to enable analytic service. Requires an account, which can be obtained for free at http://analytics.google.com. More information can be found below.
  • Enable Syntax Highlighting
    If true, will enable syntax highlighting of code blocks automatically within posts. Disable by settings value to false.
If you misconfigure you can replace the /assets/js/global.min.js with the original file to restore defaults.

Feature Configuration & Use

Featured Posts

Astro has two different post styles, its standard minimal post style which is great for smaller blog posts, and its Featured post style which is aimed at large articles, guides, and other documents. Featured blog posts have an adjusted header, which displays a large cover image and the title of the post. This image can be customized by the user, or left blank for the default gradient background.

Creating a featured post is simple, first create your standard blog post and add your content. Within the post markdown editor add the following code at the top ![featureimage]. This will create an image upload area in the live preview section of the editor. Upload the image you wish to use as the featured background. For improved performance its recommended that any large images are reduced in size (1680 width is recommended) and compressed. Below is an example of how the code might look in your editor.

This is my first paragraph of text.
![featureimage]

Once you have added content to your post, and a feature image, publish the post. Return back to the posts index, and click the star on the submitted blog post. The post is now featured and will contain a full page header. Previous posts can by also clicking the star icon, and adding an image.

Social Network Profiles

Within each blog post is an author section which contains the posters information (this can be set in the user settings of the ghost admin dashboard). Astro includes support for linking to social platforms such as facebook, twitter, github etc. within this section. Configure your social links by editing the file Astrotheme/post.hbs, and locate the code which is shown below.

{{! ADD AUTHOR SOCIAL LINKS HERE }}
{{! REMOVE ANY UNWATED LINKS }}
{{! ADD AUTHOR SOCIAL LINKS HERE }}
{{! TWITTER }}<li><a href="http://twitter.com/" class="twitter" target="_blank"><i class="fa fa-twitter"></i></a></li>
{{! FACEBOOK }}<li><a href="http://facebook.com/" class="facebook" target="_blank"><i class="fa fa-facebook"></i></a></li>
{{! GITHUB }}<li><a href="http://github.com/" class="github" target="_blank"><i class="fa fa-github-alt"></i></a></li>
{{! RSS - LEAVE DEFAULT}}<li><a href="{{@blog.url}}/rss/" class="rss" target="_blank"><i class="fa fa-rss"></i></a></li>
{{! YOUTUBE }}<li><a href="http://youtube.com/" class="youtube" target="_blank"><i class="fa fa-youtube"></i></a></li>
{{! DRIBBBLE }}<li><a href="http://dribbble.com/" class="dribbble" target="_blank"><i class="fa fa-dribbble"></i></a></li>
{{! GOOGLE+ }}<li><a href="http://plus.google.com/" class="googleplus" target="_blank"><i class="fa fa-google-plus"></i></a></li>
{{! INSTAGRAM }}<li><a href="http://instagram.com/" class="instagram" target="_blank"><i class="fa fa-instagram"></i></a></li>
{{! LINKEDIN }}<li><a href="http://linkedin.com/" class="linkedin" target="_blank"><i class="fa fa-linkedin"></i></a></li>

Edit each lines href="" value, including your profile link for that particular platform. Delete any lines for platforms which are not wanted. They can be added again later at any time by copying them from the original source file. Astro contains 2 sets of social network links, both of these need to edited, or removed if unwanted.

Static Pages

Ghost release 0.4.0 added support for static pages which can be added to your blogs navigation. In order to create a static page create a new blog post with the pages content and header. Once created click the post settings icon and tick to enable the Static Page option. Copy the post URL which is shown in the same window. This URL needs to be added to the navigation section of the theme files. Open up Astrotheme/default.hbs and edit the navigation sections to include the link to your page and a desired name. Below is an example of the navigation section. Reference Lines 30 & 44.

{{! ADD STATIC PAGE LINKS HERE }}
{{! REMOVE THE THREE DEFAULT LINKS BELOW IF NOT USED }}
{{! REPEAT BY ADDING LINKS TO THE FOOTER BELOW AS COMMENTED}}
<li><a href="/about/">About</a></li>
<li><a href="/portfolio/">Portfolio</a></li>
<li><a href="/contact/">Contact</a></li>

Disqus Comments

Setting up Discus comments for individual posts is simple. Firstly you will need an Disqus account, which can be obtained freely at http://disqus.com. Once you have created an account and setup your shortname, enter it into the theme configuration which can be found within the /assets/js/global.min.js file.

Once you have setup and enabled comments, all administration for comments can be done via the Disqus control panel. To access this admin panel navigate to Disqus, and select 'Admin' located in the header of the page. If at any time you wish to disable comments, you can do this by deselecting the 'Use Disqus Comments' theme option.

Google Comments

If you prefer, you can enable Google Plus comments to replace Disqus comments. To do this enable the setting in the configuration file located at /assets/js/global.min.js. If a Disqus shortname is set, this will override Google Comments, so set Disqus to blank ''. Please note, Google Plus Comments are unsupported by Google, and I recommend that Disqus is used instead. The implementation used to integrate Google Comments is not documented or supported by them, and may stop working without notice at any time.

Google Prettify

Astro comes with the Google Prettify plugin preinstalled. This plugin will syntax highlight and format any code which you post in your blog. A code block can be created by tabing in a section of code twice in the markdown editor, or by 4 spaces. Too disable this feature edit the theme configuration file located at /assets/js/global.min.js.

// THIS IS AN EXAMPLE OF JS SYNTAX HIGHLIGHTING

Post Annotations

Within your blog posts you can include annotations for paragraphs of text, images, and code. To do this requires using some HTML markup inside the post editor. You can copy and paste the section of code below into your editor and replace the comments with the desired content. If you are annotating code sections, you may need to wrap the code in <code> and <pre> tags.

<div class="container">
ADD YOUR MAIN CONTENT HERE E.G. Image, Code.
<div class="reference">
<strong>Annotation Title Here</strong>
<p>Annotation text here.</p>
</div>
</div>

Google Analytics

Astro supports Google Analytics on all pages. Enabling this feature requires an active account with the Google Analytics service (http://analytics.google.com). Once an account is created, and website registered, the UID code is required which is a similar format to the following: UA-28282324-1. Copy this code into the configuration file which can be found at assets/js/global.min.js.
